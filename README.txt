# 评分系统使用说明

## 📦 一键启动

### 方式一：直接运行 EXE（推荐）
- **文件**: `web_app/dist/ScoringSystem/ScoringSystem.exe`
- **功能**: 双击即可启动，自动打开浏览器

### 方式二：使用批处理脚本
- **文件**: `一键启动评分系统.bat`
- **功能**: 启动服务器后自动打开浏览器

### 方式三：简单启动
- **文件**: `启动评分系统.bat`
- **功能**: 仅启动服务器，需手动打开浏览器

---

## 🌐 访问地址

### 本机访问
http://localhost:5000

### 局域网访问（其他设备）
将 IP 替换为运行服务器的电脑 IP：
http://[服务器IP]:5000

## ⚙️ 功能列表

| 功能 | 说明 |
|------|------|
| 🎯 创建比赛 | 设置比赛名称、组数、评委数、选手数 |
| ⚖️ 评委权重 | 支持小数权重设置，最终得分=Σ(评委打分×权重) |
| 📥 Excel导入 | 导入评委、组别、选手、评分项名称 |
| ⭐ 星级评分 | 1-5星评分，实时计算总分 |
| 📊 统计分析 | 加权平均分汇总，支持导出Excel |
| 📤 批量导入 | 支持导入评分成绩 |

## � 打包目录说明

`web_app/dist/ScoringSystem/` 目录包含完整的可执行程序：
- `ScoringSystem.exe` - 评分系统主程序
- `scoring_web.db` - SQLite数据库（包含比赛数据）
- `_internal/` - 程序依赖库

## ❓ 常见问题

### Q: 局域网其他设备无法访问？
A: 请检查防火墙设置，允许5000端口入站

### Q: 权重设置有什么要求？
A: 权重总和需要等于100%，支持小数

### Q: 数据会丢失吗？
A: 数据保存在 `scoring_web.db` 中，只要不删除该文件就不会丢失

### Q: 如何更新数据？
A: 直接运行 `ScoringSystem.exe` 即可使用最新数据

---

## 📞 技术支持

如有问题，请检查：
1. 是否有管理员权限运行程序
2. 端口5000是否被占用
3. 确保 `scoring_web.db` 与 `ScoringSystem.exe` 在同一目录
